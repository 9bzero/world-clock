# Contributing

Thank you for your interest in contributing! All kinds of contributions are welcome.

## Setup

1. Fork and clone the repository
2. Run `npm install` to install dependencies
3. Run `npm run dev` to start the development server

## Workflow

1. Create a new branch for your feature or fix
2. Make your changes — keep them focused and minimal
3. Use conventional commit messages: `feat:`, `fix:`, `docs:`, `refactor:`
4. Open a Pull Request with a clear description of what you changed and why

## Guidelines

- One feature or fix per PR — keep changes scoped
- Follow the existing code style (TypeScript, ESLint)
- Add comments for non-obvious logic
- Test your changes locally before opening a PR

## Reporting Issues

Open an issue with:
- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior

---

Questions? Open an issue and I will get back to you.
